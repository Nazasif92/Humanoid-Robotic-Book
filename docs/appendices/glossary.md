# Appendix C: Glossary

This appendix contains definitions for key terms used throughout the book.

## A

**Action**: A ROS 2 communication pattern for long-running tasks that may need to be monitored or canceled.

## D

**DDS**: Data Distribution Service - the middleware that underlies ROS 2 communications.

## G

**Gazebo**: A physics-based 3D simulation environment used for robotics applications.

## L

**Launch File**: A configuration file that allows you to start multiple ROS 2 nodes at once with predefined parameters.

## N

**Node**: A single executable process that performs a specific task in ROS 2.

## Q

**QoS**: Quality of Service - settings that control communication behavior between ROS 2 nodes.

## R

**ROS 2**: Robot Operating System 2 - a flexible framework for writing robot software.

**RViz**: 3D visualization tool for displaying ROS 2 data.

## S

**Service**: A ROS 2 communication pattern for request-response interactions.

## T

**Topic**: A ROS 2 communication channel where nodes publish and subscribe to messages.

## U

**URDF**: Unified Robot Description Format - an XML format for representing robot models.

## V

**VLA**: Vision-Language-Action - a paradigm for multimodal autonomous systems.

**VSLAM**: Visual Simultaneous Localization and Mapping.

## X

**Xacro**: XML Macros - a macro language for XML that extends URDF.
